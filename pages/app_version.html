Application Version

Application version page is where you can configure and upload your content before releasing it.

Importing Content

A version requires content files. To upload the files, you may click on the Import Content button. Depending on your account features, you may be asked if you want to import the content using a ZIP file or by choosing another app’s version.

Importing using a ZIP file

To import a version using a ZIP file, you need to prepare a valid zip file first. It’s pretty , and in most cases, it requires packing your application’s directory into a ZIP using built-in or 3rd party archiving applications. You can learn more about valid zip file structure TODO here.

Version Properties

Executable File

Allows you to select which executable file should be executed by the Patcher. The list includes only a set of files that are recognized as a valid executable for a target file system. That means that those will be *.exe files for Windows, *.app bundles for OSX and executable ELF binaries for Linux.

When a new version content is sent, PatchKit sets the executable file to either:

  • The same that was set for the previous version if exists
  • The first executable file found in the content files

Changing the executable file have an immediate effect and can be modified even for alredy published releases.

Execution Arguments

List of arguments passed to the executable file at the launch. You may pass any number of arguments to change the behavior of the application. It’s usable if you need to dynamically change the behavior of your application. It’s especially usable if you distribute your application on multiple channels.

Creating a new draft version copies execution arguments from the previous one, so you don’t need to set it every time a new version is released.

Changing the execution arguments have an immediate effect and can be modified even for already published releases.
