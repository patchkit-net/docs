While PatchKit desktop applications are based on the fail-safe architecture principles, there are several cases when something may go wrong during the initial installation or patching stage. On this page, you can learn what common issues your users may encounter.

Downloading Issues

Downloading issues are the most common ones. While worldwide Internet infrastructure is getting better every day, there’s still plenty of things that may slow down or completely prevent the user from downloading the application.

Possible Reason: ISP Hardware Issues

You should ask your user to check the Internet connection quality first. Speed tests are useful, but some ISPs are configuring separate routing rules to popular speed tests sites so that the results can be better than usual.

  • Ask the user to visit http://benchmark.patchkit.net/. See if results to their nearby servers are looking good.
  • Ask the user to perform ping test on this page: https://freeola.com/line-test/ high jitter or packet loss value may be the reason
  • If the user has a secondary internet connection, it should be examined

Possible Reason: Wi-Fi

Wi-fi hardware often causes trouble. It’s hard to find a router that does not need to be reset once in a while. That’s why you need to ask your user to try and reset the Wi-fi router in use. Trying to connect via cable connection is recommended.

Possible Reason: Routing Issues

When connecting from point A to B, there’s plenty of cables and hardware in between. The routing configuration of each of those is deciding where the connection should go through, and sometimes it may go through a suboptimal way. It can even fall-back to a slower route due to incidents like a power outage in a strategic point.

  • Ask the user to perform a benchmark http://benchmark.patchkit.net/ to see if the problem is with one or more mirrors.
  • Ask the user if the problem affects other services. If it does, the user should contact the ISP.

Possible Reason: Malicious or Packet Inspection Software

We’ve observed that in some configurations, software that is tempering with the Internet connection may slow it down or prevent the user from downloading specific data.

  • Ask the user to try installing the application on another machine or operating system if possible
  • Ask the user to perform anti-virus scan and anti-malware scan. We recommend Malwarebytes scanning software.
  • Ask the user to disable the anti-virus during the downloading procedure.

Possible Reason: Firewall Configuration

Depending on your user machine, there may be a firewall installed with the rules that effectively prevent downloading the data from untrusted sources.

If the user is trying to install the software…

  • …on home PC, we suggest disabling the firewall.
  • …at workplace, we suggest contacting the administrator.
  • …at school, we suggest contacting the teacher or the administrator.

Nothing Helps?

When the user had tried everything but the issue is still there:

  1. Ask for launcher-log.txt and output_log.txt files. Check here where to find those.
  2. Ask the user to visit http://benchmark.patchkit.net and share the results using the share link.
  3. Contact PatchKit’s support by email providing all files mentioned above, along with your PatchKit’s account e-mail address and application’s name.

Installation / Patching Issues

PatchKit packages are encrypted, compressed packages, so they need to be decrypted, unpacked, and saved to the disk drive. This process is called the installation process, and there are several places where it can fail.

Possible Reason: Anti-Virus Software

Anti-virus software is meant to protect the user from installing any malicious software, but it can act extra-careful with launchers. While our launcher is doing its best to co-exist with most of the anti-viruses, it still can get in conflict with some of them.

Ask the user to disable the anti-virus software during the process of installation.

Possible Reason: Malicious Software

Malicious software can hook-up to the system events in a similar manner to anti-virus software. These are often badly-written and may tamper with the operating system’s stability.

Ask your user to perform a system scan using Malwarebytes free scanner.

Possible Reason: Hardware Issue

Faulty hardware like hard disk, RAM, or even CPU can affect the installation process. It is worth to ask the user if there were any strange issues with the PC lately. We recommend reading this article.

Nothing Helps?

  1. Ask for launcher-log.txt and output_log.txt files. Check here where to find those.
  2. Contact PatchKit’s support by e-mail providing all files mentioned above, along with your PatchKit’s account e-mail address and application’s name.

Application Running Issues

When the installation goes smoothly, but the application is not starting, do not panic. These issues are mostly easy to solve.

Possible Reason: Wrong Executable File

When your game is first uploaded, PatchKit is trying to guess which executable file is the main application file. Make sure to verify it.

  1. Navigate to https://panel.patchkit.net.
  2. Log in into your account if needed.
  3. Navigate to your application’s latest published version.
  4. See if the Executable File is set correctly.

Possible Reason: Missing Application Files

While Patcher is making sure that all provided files are installed in the system, sometimes the application package may be incomplete. The usual reason is a human mistake during the packaging process. Make sure to:

  • Zip the whole folder with the application. It’s OK to have one directory on the top-level, and application executable in that directory.
  • Use continuous-integration tools whenever possible. We recommend integrating with patchkit-tools as well.

Possible Reason: Anti-Virus Quarantine

Sometimes Anti-virus software can quarantine your game files, restricting any access. Ask your user to disable anti-virus software before running your application. If it doesn’t help, ask the user to check the quarantine list.

Possible Reason: Malicious Software

Because of the variety of this kind of applications, there’s a possibility that there’s an interference. Ask the user to scan the system. We recommend Malwarebytes scanner.

Nothing helps?

There may be a plenty of reasons why your application may not be working correctly. If you’re out of options, try looking for any error messages and search it on the web. You can always contact our support for the advice.
